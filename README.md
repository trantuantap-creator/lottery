# 🎡 宝贝の幸运大转盘

一个写给女朋友的抽奖大转盘小网页，纯静态、零依赖、可直接部署到 GitHub Pages。

> 在线访问：https://trantuantap-creator.github.io/lottery/

## ✨ 特性

- **华丽视觉**：渐变星空背景 + 飘浮爱心 + 旋转光环 + 24 个闪烁灯泡 + 金边描线
- **真实抽奖体验**：转盘惯性减速 + 随扇区滴答音 + 中奖彩带飞舞 + 弹窗动画
- **Web Audio 合成音效**：滴答声、上扬琶音中奖音、下行失落音，全部实时合成（零音频文件依赖）
- **右上角静音开关**：状态记 localStorage，下次访问保留
- **结果持久展示**：抽过之后，信息条会一直显示「今日抽到：xxx」
- **一天一次限制**：基于客户端 IP + localStorage（详见下方"实现说明"）

## 🚀 部署到 GitHub Pages

1. 推送代码到 `main` 分支
2. 仓库 **Settings → Pages**
3. **Source** 选 `Deploy from a branch`，**Branch** 选 `main` / `/(root)`
4. 等 1-2 分钟生效

## 🧠 实现说明

### 概率与扇区分离设计
- 转盘上**显示**很多奖品（让视觉更丰富），但**真正能抽中**只有 3 个
- 抽奖逻辑：先用 `Math.random()` 按 20% / 5% / 75% 选出结果类型，再随机挑该类型的一个扇区作为停止位
- 转盘旋转角度反推自目标扇区，确保指针落点准确

### IP 限制（一天一次）
1. 调用 `api.ipify.org`（备选 `ipapi.co`）拿到访客公网 IP
2. localStorage key 用 `lottery_${ip}_${YYYY-M-D}` 拼接
3. 抽完写入结果对象 `{ type, date, at }`
4. 跨天自动失效（日期变了 key 就变了）

⚠️ **已知漏洞**：GitHub Pages 是纯静态托管，无法做服务端校验。换浏览器、清缓存、隐身模式、换网络都能再抽一次。对女朋友够用 😄；如需真·防作弊请加 Cloudflare Workers / Vercel Functions 等后端。

### 音效合成
全部用 Web Audio API 的 `OscillatorNode` 实时合成：
- **滴答**：1500Hz → 800Hz 方波，50ms 衰减
- **中奖**：C5-E5-G5-C6-E6 三角波琶音 + 后置水晶高音
- **失败**：G4-F4-D4 锯齿波下行三音
